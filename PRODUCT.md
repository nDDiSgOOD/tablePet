# Product

## Register

product

## Users

TablePet 是 **极客 / 开发者自用** 的桌面伴侣 dashboard。

典型用户画像：在自己电脑上跑 LLM 项目的工程师，习惯了 Linear / Raycast / Notion Calendar 的克制；同时不抗拒"养电子宠物"的小情绪，会在调试 ESP32 串口的间隙看一眼桌宠状态。他们会自己改 prompt、看 token、调模型 —— 但**不希望 dashboard 被这些"工程感"占满，留一点伙伴的呼吸空间**。

使用情境：

- 在 IDE 旁边开着一个浏览器 tab 看桌宠
- 主要任务是「**和宠物对话** / 看今天聊了什么 / 调一下账户和模型 / 切人格**」**
- 偶尔会插入 ESP32 串口调试、查 LLM 余额这种工程操作

## Product Purpose

一个能落地在本机、被开发者长期使用、有"陪伴感"但不"幼稚"的桌宠 dashboard。

成功的样子：用户开着它一周不烦，每天会有一两次因为「今日日记 / 桌宠心情 / 账户余额提示」**想多看一眼**，而不是只在调试时打开。

## Brand Personality

**克制 · 温柔 · 像伙伴。**

- **克制**：不堆功能、不炫酷动画、不靠 emoji 卖萌；信息密度高的地方就老老实实排版。
- **温柔**：暖色深色调（不是冷调机房黑），圆角不极端、对比度不刺眼，hover/focus 都是慢慢亮起来。
- **像伙伴**：状态文案不是冷冰冰的"Token used: 12,345"，是"今天我们聊了 12 句话哦"——拟人、软、低调，不喧宾夺主。

## Anti-references

- **Bootstrap 模板感**：千篇一律的卡片栅格 + 渐变按钮 + 标准蓝灰配色。
- **微信小程序 / 小米加湿器**：过度卡通化、大色块圆角按钮、emoji 堆砌。
- **Notion 那种通用白板**：功能堆叠、缺少角色感、"像个文档工具"。
- **ChatGPT / Claude 的纯聊天界面**：丢掉了"宠物"的存在感，整个就是一个对话框。

## Design Principles

1. **桌宠是主角，仪表是配角**：宠物名字、心情、人格永远在视觉的"上席"；token / API / 模型这种工程数据放第二动线。但不要因此把工程数据藏起来——开发者要能 1 秒看到。
2. **暖灯火，不机房**：深色但是暖的（OKLCH hue 偏 60-90 而非 250）。所有"被点亮"的元素（active 状态、hover、当前选中人格）应该像桌灯被点开，而不是 LED 闪。
3. **节奏靠留白，不靠分割线**：参考 Apple Calendar / Notion Calendar 的处理。卡片之间用空气隔开，而不是 1px border + box-shadow 的"幽灵卡"。
4. **文案永远以"我们"为主语**：状态、tooltip、空态、错误提示，第一人称是桌宠，第二人称是用户。"今天我们聊了 12 句"，不是"对话总数：12"。
5. **像 IDE 一样可信，像伙伴一样不抢戏**：余额、模型、API key 这些数字必须**精确显示**（不四舍五入、不模糊化），但呈现方式要轻——细字号、低饱和、靠右对齐。

## Accessibility & Inclusion

- WCAG AA：正文 ≥ 4.5:1，大字 ≥ 3:1。**目前 dashboard 的 muted 灰色文本在暖深色底上需要复核**（典型的 AI 设计高发失败点）。
- 支持 `prefers-reduced-motion`：所有人格切换 / 状态变化的动画必须有 reduce-motion 替代（淡入或瞬时切换）。
- 单语：当前仅中文，但 UI string 不混用全角半角，标点统一中文（避免出现 `Token: 12,345` 和 `余额：$0.94` 风格混乱）。
- 色觉无障碍：心情用文字 + 颜色双重标识，不只用红/绿区分（已有 emoji，方向是对的，但需要确保不依赖颜色单通道）。
